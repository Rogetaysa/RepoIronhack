Comprensin del Clculo del Efecto Mnimo Detectable (EMD)

En el contexto de las pruebas A/B, especialmente en el comercio minorista o cualquier otro dominio, determinar el Efecto Mnimo Detectable (EMD) es crucial para disear experimentos que sean prcticos e informativos. El EMD nos indica el tamao del efecto ms pequeo que nuestra prueba est capacitada para detectar dado un tamao de muestra especfico, un nivel de significancia ((\alpha)), y un poder estadstico ((1 - \beta)).

El Proceso Explicado

  1. Inicializacin: Comenzamos con un conjunto de datos y decidimos las fracciones de este conjunto que queremos probar. Estas fracciones varan del 5% al 50% en incrementos del 5%. Para cada fraccin, nuestro objetivo es calcular el EMD para entender cmo cambiar el tamao de nuestra muestra afecta nuestra capacidad para detectar diferencias significativas entre dos grupos (por ejemplo, un grupo de control y un grupo de tratamiento).

  2. Bucle de Simulacin: Para cada fraccin:

    • El conjunto de datos se divide en un grupo de prueba y un grupo de control basado en el tamao actual de la fraccin.
    • Realizamos esta divisin 30 veces para mitigar el efecto de la aleatoriedad y tomar el promedio, asegurando que nuestros clculos de EMD sean robustos y confiables.

  3. Clculo del EMD:

    • Para cada divisin, calculamos la desviacin estndar de los grupos de prueba y control y luego calculamos una desviacin estndar agrupada. Esta desviacin estndar agrupada ayuda a estimar la desviacin estndar comn entre los dos grupos como si fueran un solo grupo.
    • El EMD se calcula utilizando la frmula proporcionada a continuacin. Esta frmula considera el nivel de significancia, el poder deseado de la prueba y la desviacin estndar agrupada para calcular el tamao del efecto ms pequeo que la prueba puede detectar de manera confiable.

  4. Visualizacin:

    • Despus de calcular el EMD para cada fraccin, graficamos estos valores contra el tamao de la muestra. Esta representacin visual ayuda a entender la relacin entre el tamao de la muestra y la detectabilidad de los efectos, guiando la planificacin de futuras pruebas A/B.

La Frmula para el EMD

El EMD se calcula usando la siguiente frmula:

$$ \text{EMD} = (z_{\alpha/2} + z_{\beta}) \times \text{Desviacin Estndar Agrupada} \times \sqrt{\frac{2}{n}} $$

Donde:

  • (z_{\alpha/2}) es el valor crtico de la distribucin normal estndar para una prueba de dos colas al nivel de significancia (\alpha).
  • (z_{\beta}) es el valor crtico que corresponde al poder deseado ((1 - \beta)).
  • La desviacin estndar agrupada es una estimacin de la desviacin estndar comn de los dos grupos.
  • (n) es el tamao de la muestra del grupo de control.

  • Comprender la relacin entre el tamao de la muestra y el EMD puede ayudar a empresas e investigadores a disear experimentos ms eficientes y efectivos.

# Initialize variables
contador = 0  # This will track the fraction of the dataset allocated to the test group
data = {}  # To store results

# Loop to calculate MDE for splits ranging from 5% to 50% of the dataset
while contador <= 0.45:
    contador += 0.05  # Increment the fraction for each iteration
    mde_list = []  # To store MDE values for each iteration
    avg_list = []  # To store average values for the control group
    
    # Perform 30 simulations for each split to average the MDE
    for i in range(1, 31):
        # Randomly split the dataset into test and control groups based on the current fraction
        test_df, control_df = train_test_split(df, test_size=contador)
        
        # Define the parameters for the MDE calculation
        alpha = 0.05  # Significance level
        beta = 0.8    # Desired statistical power
        
        # Calculate standard deviations for both groups
        std_dev_test = np.std(test_df['price'])
        std_dev_control = np.std(control_df['price'])
        
        # Calculate sample sizes
        sample_size_control = len(control_df)
        sample_size_test = len(test_df)
        
        # Calculate critical values for the two-tailed test
        z_alpha = norm.ppf(1 - alpha/2)
        z_beta = norm.ppf(beta)
        
        # Calculate pooled standard deviation
        pooled_std_dev = np.sqrt(((sample_size_control - 1) * std_dev_control**2 + 
                                  (sample_size_test - 1) * std_dev_test**2) / 
                                 (sample_size_control + sample_size_test - 2))
        
        # Calculate the MDE
        mde = (z_alpha + z_beta) * pooled_std_dev * np.sqrt(2 / sample_size_control)
        
        # Append the MDE and average control group price to their respective lists
        mde_list.append(mde)
        avg = np.mean(control_df['price'])
        avg_list.append(avg)
    
    # Calculate and store the average MDE, average price, and MDE percentage for the current split
    data[(round(contador, 2))] = {'mde': round(np.mean(mde_list), 4),
                                  'avg': round(np.mean(avg_list), 4),
                                  'mde_%': round(np.mean(mde_list) / np.mean(avg_list) * 100, 2)}

Despus de dividir nuestro conjunto de datos en grupos de control y prueba, nos centramos en comparar sus distribuciones para evaluar si difieren significativamente. Este paso es crucial para validar las suposiciones de nuestro marco de prueba A/B y asegurar la fiabilidad de nuestros resultados.

Realizando la Prueba de Kolmogorov-Smirnov (KS) Con nuestros datos seleccionados, procedemos a realizar la prueba de Kolmogorov-Smirnov (KS). Esta prueba no paramtrica evala la hiptesis de que las distribuciones de dos muestras son iguales. Es particularmente til en las pruebas A/B para comparar las distribuciones de resultados de nuestros grupos de control y de prueba sin hacer suposiciones sobre la distribucin subyacente de los datos.

La prueba KS proporciona dos piezas clave de informacin:

  • La estadstica KS: Una medida de la distancia entre las funciones de distribucin empricas de las dos muestras.
  • El valor p: Indica la probabilidad de observar los datos asumiendo que la hiptesis nula es cierta. Un valor p bajo (tpicamente < 0.05) sugiere que podemos rechazar la hiptesis nula y concluir que las distribuciones de las dos muestras son significativamente diferentes.
statistic, p_value = ks_2samp(data1, data2)
print(f"KS statistic: {statistic}")
print(f"P-value: {p_value}")

El resultado de la prueba KS nos ayuda a entender si las intervenciones o cambios que estamos probando tienen un impacto estadsticamente significativo en la distribucin de nuestra mtrica de inters, permitindonos tomar decisiones informadas basadas en los resultados de nuestra prueba A/B.

En este paso, estamos realizando una prueba de equivalencia entre nuestros grupos de control y prueba para determinar si sus medias son estadsticamente equivalentes dentro de lmites predefinidos. La prueba de equivalencia es crucial para situaciones en las que queremos confirmar que dos grupos no difieren por una cantidad significativa, en lugar de simplemente buscar cualquier diferencia.

  • Definiendo los Lmites de Equivalencia: Los lmites de equivalencia (low y upp) definen el rango dentro del cual las medias de los dos grupos deben estar para considerarse equivalentes. En este caso, los lmites estn establecidos en -0.5 y 0.5, lo que significa que estamos probando si la diferencia en las medias es significativamente menor que 0.5 unidades y ms que -0.5 unidades en cualquier direccin.

  • Realizando la Prueba de Equivalencia: Utilizamos el mtodo ttost_ind de la biblioteca statsmodels, que realiza Dos Pruebas Unilaterales (TOST) para las medias de dos muestras independientes. Este mtodo devuelve tres valores:

    • pv1: El valor p de la prueba del lmite inferior, que prueba si la diferencia de medias es significativamente mayor que el lmite inferior.
    • pv2: El valor p de la prueba del lmite superior, que prueba si la diferencia de medias es significativamente menor que el lmite superior.
    • result: Un valor booleano que indica si se cumplen ambas condiciones y, por lo tanto, si las medias son equivalentes dentro de los lmites especificados.

  • Interpretando los Resultados: La salida nos proporciona los valores p para ambas pruebas, inferior y superior. Si ambos valores p estn por debajo de un nivel de significancia elegido (comnmente 0.05), podemos concluir que las medias de los dos grupos son estadsticamente equivalentes dentro de nuestros lmites definidos.

Este enfoque permite una comprensin matizada de las diferencias entre los grupos de control y prueba, centrndose en la equivalencia prctica en lugar de solo la diferencia estadstica.

Calculando el Efecto Mnimo Detectable (MDE) Temporal

Al disear pruebas A/B o cualquier experimento basado en datos, entender el Efecto Mnimo Detectable (MDE) a lo largo del tiempo puede ser crucial. El MDE representa el tamao del efecto ms pequeo que la prueba tiene la capacidad adecuada para identificar. A medida que los datos se acumulan y las distribuciones se estabilizan con el tiempo, el MDE puede disminuir. Esto significa que efectos que no pueden ser detectables con un conjunto de datos ms pequeo (por ejemplo, datos semanales) podran hacerse evidentes al analizar un conjunto de datos ms grande recopilado durante un perodo ms largo (por ejemplo, datos mensuales o trimestrales).

Entendiendo el Enfoque

En un escenario donde se rastrea el comportamiento del consumidor a lo largo del tiempo, como los hbitos de compra de un producto especfico (por ejemplo, detergente), es comn no observar cambios significativos en marcos de tiempo cortos como una semana. Sin embargo, durante un mes o ms, los patrones de compra pueden emerger ms claramente.

Para acomodar este aspecto temporal en nuestro anlisis, nuestro conjunto de datos evoluciona para incluir mltiples columnas, cada una representando el promedio acumulativo de consumo sobre perodos de tiempo crecientes. Por ejemplo:

  • La Columna 1 podra representar el consumo promedio de enero.
  • La Columna 2 podra representar el promedio acumulativo de enero y febrero.
  • La Columna 3 incluira entonces el promedio acumulativo de enero, febrero y marzo, y as sucesivamente.

Esta estructura nos permite observar cmo se estabiliza o cambia el consumo promedio a medida que se acumulan ms datos con el tiempo.

Reaplicando el Clculo del MDE

Con esta estructura de datos temporal, podemos reaplicar nuestro mtodo de clculo del MDE. Iterando a travs de las columnas (cada una representando promedios acumulativos sobre diferentes perodos de tiempo), realizamos el clculo del MDE para cada segmento temporal. Este proceso iterativo es similar al enfoque original donde calculamos el MDE para diferentes tamaos de muestra. Sin embargo, en este contexto, estamos evaluando cmo la extensin temporal de nuestros datos afecta el MDE.
